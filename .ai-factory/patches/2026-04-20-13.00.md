# GitHub Actions workflow artifact naming issue

**Date:** 2026-04-20 13:00
**Files:** .github/workflows/release.yml
**Severity:** medium

## Problem

Release workflow was producing archives with generic name `NetBox.7z` instead of platform-specific names like `NetBox.ARM64.24.12.2.608.163.7z`.

## Root Cause

The GitHub Actions variable interpolation `${{ matrix.platform }}` and `${{ env.version }}` in the `echo ... >> "$GITHUB_ENV"` command may not have been correctly interpolated when writing to GITHUB_ENV.

## Solution

Added explicit debug output to verify variable values at runtime. This helps diagnose if variables are being set correctly while maintaining the same artifact naming logic.

## Prevention

- Add debug output when troubleshooting GitHub Actions variable issues
- Use explicit debugging with `echo "DEBUG: ..."` to verify variable interpolation
- In GitHub Actions, always add debugging to understand variable passing between steps
