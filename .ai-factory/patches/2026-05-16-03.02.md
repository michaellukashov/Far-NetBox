# Fix: "Index is out of range" crash when importing .netbox sessions with empty names

**Date:** 2026-05-16 03:02
**Files:** 
- src/core/NamedObjs.cpp
- src/core/HierarchicalStorage.cpp
- src/NetBox/XmlStorage.cpp
- src/NetBox/WinSCPFileSystem.cpp
- src/NetBox/WinSCPPlugin.cpp
**Severity:** high

## Problem

Importing a `.netbox` session file caused a fatal exception: `Index is out of range`. The crash occurred when the XML file contained a session element with an empty or missing `name` attribute, or when the `Session` value read from the file was empty.

Error: `Index is out of range`
File: `T:\scp-user@10.147.19.230.netbox`

## Root Cause

`TNamedObject::SetName()` unconditionally called `UnicodeString::SubString(1, 3)` on the session name to check for the hidden prefix (`_!_`). When the session name was empty (length 0), `SubString(1, 3)` failed the 1-based bounds check (`Idx > Length`) and threw `Exception("Index is out of range")`.

The empty session name could originate from:
1. A malformed `.netbox` XML file with a `<Session>` element lacking a `name` attribute.
2. A corrupted or hand-edited export file.

Additionally, `UnMungeStr()` and `PuttyUnMungeStr()` were not defensive against empty strings, and the XML storage's `DoGetSubKeyNames()` added empty strings to the key list without filtering, propagating the empty value all the way to `SetName()`.

## Solution

1. **Guard `SetName` against empty strings** (`src/core/NamedObjs.cpp`):
   Added `!Value.IsEmpty()` before the `SubString` call to prevent the out-of-range access.

2. **Make `UnMungeStr` empty-safe** (`src/core/HierarchicalStorage.cpp`):
   Added an early return for empty input strings to avoid unnecessary PuTTY `unescape_registry_key` calls and potential downstream issues.

3. **Filter empty subkey names in XML storage** (`src/NetBox/XmlStorage.cpp`):
   `DoGetSubKeyNames()` now skips elements whose `name` attribute or text content is empty, preventing empty session names from reaching the session list at all.

4. **Add diagnostic logging around import paths** (`src/NetBox/WinSCPFileSystem.cpp`, `src/NetBox/WinSCPPlugin.cpp`):
   Wrapped `Load()` calls in try-catch blocks with `[FIX]`-prefixed `AppLogFmt` logs. Added an explicit empty-session-name check with logging in the `OPEN_ANALYSE` path to provide clear diagnostics instead of a cryptic bounds exception.

## Prevention

- Always guard 1-based string indexing (`SubString`, `operator[]`) with `IsEmpty()` or length checks.
- Treat external data (imported XML, registry, INI) as untrusted: validate presence and non-emptiness of required fields before using them in string operations.
- When fixing a bounds crash, audit callers of the crashing function for missing empty-string handling.
